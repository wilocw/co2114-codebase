1.2 Facility Placement Problem

In this lab, you will be designing agents that solve a facility placement problem.

The facility placement problem describes the idea of placing facilties in locations that optimise some utility or cost function.

Facility placement is an example of an NP-hard problem, where it can be very difficult to solve. The search space involves moving facilties step-by-step and recalculating the utility.

We will implement local and global local search optimisers to try and find the optimal placement for hospitals such that the placement minimises the distance between houses.

Percepts

As we are using local search approaches, we don't need an environment that represents the entire search space as graph. At each discrete interval in the optimisation, the agent needs only information about its current state and its neighbors.

def percept(self, agent):
    return self.state, self.neighbours

This is also why we can use XYEnvironment, because neighbours are defined using the tiles immediately adjacent to hospital positions

State

We define our state as a data structure containing the locations of each hospital: a dictionary with the key-value pairs comprising a Hospital instance and its x,yx,yx,y location in the environment. Likewise, we have a dictionary of House objects indexing their x,yx,yx,y locations:

@property
def state(self):
    return {
        "hospitals": {
            thing: thing.location
                for thing in self.things
                    if isinstance(thing, Hospital)},
        "houses": {
            thing: thing.location
                for thing in self.things
                    if isinstance(thing, House)},
        "bounds": {
            "xmin": 0, "xmax": self.width-1,
            "ymin": 0, "ymax": self.height-1}
    }

To ensure possible neighbours aren't defined outside of the bounds of the environment, we define these in the state.

Neighbours

Unlike in Lab 2, in this search problem, the agent itself does not have a spatial identity: its state takes the entire placement of hospitals and houses in the environment. In a given state, we can explore a possible neighbours of a state as follows:

  • All houses are in the same place
  • Exactly one hospital is placed in a valid tile adjacent to its current position.

In this case, adjacent describes the tile immediately up or down, or left or right of the current position. A valid position is one that does not have a House or Hospital in it, and is within the bounds of the environment. This check is implemented in the is_inbounds() method of the environment.

To calculate the neighoburs of a given state, we can simply iterate through the Hospital objects in the environment, and for each valid adjacent position that hospital can move to, we create a copy of the state with only the position of that hospital changed.
The implement of neighbours is defined below. T

HospitalPlacement.neighbours() is defined in optimisation.planning, you don't need to implement it yourself

@property
def neighbours(self):
    neighbours = []
    for i, hospital in enumerate(self.state["hospitals"]):
        location = hospital.location
        for x,y in [(-1, 0), (1, 0), (0, -1), (0, 1)]:
            proposal = location[0] + x, location[1] + y
            if self.is_inbounds(proposal):
                candidate = self.state.copy()
                candidate["hospitals"][hospital] = proposal
                neighbours.append(candidate)
    return neighbours

As we have seen in the definition of percept, the agent will receive the state and any valid neighbours to explore.

Executing Actions

Our possible actions in local search-based optimisation are simple. Either we explore a new state, or the optimiser has reached its goal and is done.

HospitalPlacement.execute_actions() is defined in optimisation.planning, you don't need to implement it yourself

def execute_action(self, agent, action):
    """ Execute an action """
    command, state = action
    match command:
        case "done":
            self.success = True
        case "explore":
            agent.explore(state)

Our environment will simply execute the agent's action or set the success flag to True.

To explore a state, we simply set update the locations of the hospital to reflect the new state, provided by the environment:

def explore(self, state):
    for hospital, loc in state["hospitals"].items():
        hospital.location = loc

In the first part of this lab, we will be trying to optimise the position of the hospitals such that we minimise the distance from each house to its closest hospital.

We determine this by iterating through each house and adding to our objective the distance to its nearest hotel using the Manhattan distance.

def utility(self, state):
    """ calculate distance from each hospital to houses """
    obj = 0
    houses, hospitals = state["houses"], state["hospitals"]
    for house in houses:
        dist_to_nearest = infinity   # very big
        for hospital in hospitals:
            dist = manhattan(
                houses[house],
                hospitals[hospital])
            if dist < dist_to_nearest:
                dist_to_nearest = dist
        obj += dist_to_nearest
    return -obj

Recall the Manhattan distance from Lab 03: the number of tiles in the x direction and in the y direction summed.

We return the negative of the objective, because while we are trying to minimise the objective, our agent is seeking to maximise its utility.

This is the objective we will seek to minimise with our implementations of local search agents. In 4. [Exercise] Minimising the Maximum distance you will look at optimising for a different objective.

2. [Tutorial] Local Optimisation

In this section, we will create an agent that explores the state space using hill climbing.

Hill climbing is an example of local search that "climbs a hill" by choosing the neighbour that maximises the objective. From an initial state, hill climbing seeks to step to progressively "better" states until it cannot improve its state any further

2.1 HillClimbOptimiser

We will simply need to define a class HillClimbOptimiser that inherits from HospitalOptimiser. The implementation of our agent program is relatively straightforward: we simply calculate the utility of the current state and compare it to the maximum utility of the neighbouring states. If we have found a neighbour with a higher objective, we explore that state, otherwise the agent.

Copy the following class into your co2114_lab04.py file

class HillClimbOptimiser(HospitalOptimiser):
        
    def program(self, percepts):
        state, neighbours = percepts
        objective = self.utility(state)

        print(f"{self}: current distance {-objective}")
        print(f"{self}: possible new objectives {[-self.utility(n) for n in neighbours]}")
        
        choice = self.maximise_utility(neighbours)

        if self.utility(choice) > objective:
            return ("explore", choice)
        return ("done", None)
