Understanding Floats

When working with numbers in our AI systems, we will often need to work with values that are not integers. However, representing numbers with decimal points is not as straightforward as it seems. How do you contain a number with potentially infinite decimal places within finite memory? The answer is floating point arithmetic. This is how computers deal with calculations between non-integer numbers. You will almost certainly have used used floats when programming.

There are two common types of float: single- and double-precision. In Java, these are represented by the Float and Double type, respectively. The difference is essentially in the amount of memory each one takes up: single-precision floating point numbers are 32 bits; double-precision floats are 64 bits. A Python float is double-precision. For comparison, a Python int is 32 bits. There are other ways to represent numbers, such as using NumPy.

Understanding how floating point arithmetic works is not important for this course, but you should be aware of how numbers are represented and the potential issues this can cause when developng your artificial intelligence systems.

If you are interested in the theory that drives floating point arithmetic, you can explore the additional material provided on Blackboard.

Bad Maths

Consider the following three lines of code.

: a = 0.1
: b = 0.2
: (a+b) == 0.3

What do we expect the answer to be? True, of course. If $a = 0.1$ and $b=0.2$, then $a+b = 0.3$. But if we run this code in IPython, we will see that this is not quite the case. Why? What is $a+b$?

The reason that $0.1 + 0.2 \neq 0.3$ is that nether $0.1$ nor $0.2$ can be represented exactly in binary. This means that when performing calculations with them, there is some rounding that needs to be introduced, resulting in errors in the calculations. While these errors are often very small, they can cause issues with computing logic when checking values.

Accumulating Error

In this example, we chain together several floating point calculations.

: c = 0.
: for _ in range(100):
:    c += 1/100

We expect the value here to be $c=1$, as the sum of $100$ hundredths. However, if you print the value of $c$, you will see that there it is a little over $1$. Executing the same loop but with $1,000$ iterations of c += 1/1000 will give a different result, but still not quite 1. Try with $10,000$ iterations, and with $100,000$, and see how the result is never exactly $1$. You can see why this might be a problem when designing AI systems.

Divide by Zero

Below is an example of unexpected behaviour as a result of floating point precision. As a rule, division by zero should not be possible: any number divided by zero is undefined. Some languages handle this with exceptions and errors, some with not-a-number (NaN) values.

By default, Python will raise an ZeroDivisionError exception:

: 2/0
ZeroDivisionError

However, if we divide by a not-quite-zero value, no exception is raised. Instead, we are returned a value that is unexpected.

: d = 0.1 + 0.2 - 0.3  # this should be 0.0
: 2/d

Without proper handling, issues such as this could have cause havoc with systems.

  • tuples, e.g. (a,b)
  • sets, e.g. {'a','b','c'}
  • dictionaries, e.g. {'a':1, 'b':2}

Each data structure has different properties, an overview of which is shown in the table below. It indicates whether a data structure is

  • indexable, i.e. values can be accessed by some index
  • mutable, i.e. it can be changed after it has been created
  • and whether it allows duplicates, i.e. if the same value or instance (of an object) can occur more than once
Type Ordered Indexable Mutable Allows duplicates
list Yes Yes Yes Yes

Lists

Lists are probably the most common data you will encounter when working with Python. They are the most versatile, ensuring order, indexability, and mutablity. A list does not need to contain elements of a single type, and may contain multiple instances of the same value or instance. This versatility gives few inherent guarantees about the data contained within, however.

Indexing

Lists can be indexed using the [], with the first element of a list indexed at 0. This will be familiar to you from other programming languages.

In Python, it is also possible to use negative indexing, to index from the end of a list. For example, for a list el, while el[0] is the first element; el[-1] is the last. You can keep decrementing to index from the back.

Try the following operations in ipython to ensure you are familiar with list indexing.

: el = [1,2,3,4,5]
: el[0]
: el[1]
: el[-1]
: el[-2]
: el[-len(el)]

The function len() returns the length (i.e. the number of elements) of a data structure

We can also use indexing to change the value of an element:

: el[0] = 0
: el

It is also possible to index "slices" of a list, i.e. multiple values within a range of indexes. To slice a list, you use the : with the first and last index of the range you want.

For example,

: el[1:4]

As you can see, this has given us the equivalent of [el[1], el[2], el[3]]. By default, if you do not include one, the first index will be 0, e.g. el[:4] will give the first four elements of a list. If you don't include a last index, it will include every element to the end of the list, e.g. el[1:] will return all but the first element of the list.

You can also use negative indexing in this, e.g. el[:-3] returns all but the last three elements of a list. Likewise, you can add no indexes, el[:] which will return a copy of the list el.

Appending and concatenating lists

Sometimes when we have a list, we want to expand it. Unlike arrays in other languages, we do not need to create a new list when we want to add a value. We can use the append() method to add a new value to the end of a list:

: el.append(6)
: el[-1]

If we want to concatenate two lists, we have a few options. The first is to simply iterate through one of them and add each element to the other list:

: a = [1,2,3,4]
: b = [5,6,7]
: for x in b:
:     a.append(b)
: a

This, of course, is quite a naive approach and uses more lines of code than is really necessary.

If you want to extend an array with the elements of another, you can use the extend() method, which is similar to append() but takes a list (or other iterable data structure) as input.

: a = [1,2,3,4]
: b = [5,6,7]
: a.extend(b)
: a

An alternative, if you are creating a new list from two lists, you can use the * operator to "unpack" the lists:

: a = [1,2,3,4]
: b = [5,6,7]
: [*a, *b]

This essentially works by indicating with * iterating through elements of the variable when explicitly defining the list.

It's important to familiarise yourself with the * operator for unpacking iterable data structures, as it will appear often in the labs and tutorials.

It is also possible to concatenate two lists with the + operator, but I don't recommend this because it can become confusing. If you are using NumPy, for example, + will add the values of the lists instead. This is similar to behaviour in other languages.

List Comprehension

As we have seen the tutorial, we can create data structures using for. This essentially allows us to create a list (or other data structure) from another iterable data structure.

Consider the following, where we want to create list of the squares of numbers from 0 to 9:

: el = []
: for i in range(10):
:   el.append(i**2)
: el

Here we have created an empty list and added the elements as we iterate threough the numbers 0-9.

The range() method simply creates an iterable sequence of numbers from 0 to 10 (not including 10). It is commonly used in for loops. If we wanted to turn it into a list object, we need to wrap it in a list constructor, e.g. list(range(10)). Either way, we can iterate through it.

We streamline this by creating it inine using list comprehension:

: el = [i**2 for i in range(10)]
: el

This is an effective way to create new lists based on some function, avoiding for loops. It is also possible nest loops within a list comprehension (and also create lists of lists). See below, the truth table for $P$ and $Q$:

: TF = [True, False]
: [p and q for p in TF for q in TF]

Tuples

The tuple is another common data structure. Like lists, tuples can be indexed and sliced. They are also ordered. However, unlike lists, tuples are mutable. This means they cannot be changed, and nor can the values within change. It is not possible to add to, or extend a tuple.

To create a tuple, we use the round brackets ( ) with elements seperated by a comma. For example:

: t = (0, 1, "a")
: t

A single value or variable in () is not a tuple, unless a , included, e.g. (0,)

It is possible to index a tuple to get elements in the same way as indexing lists including negative indexing and slicing. However, it is not possible to change the value of an indexed element.

Packing and unpacking

In reality, the brackets are optional. Python will be default pack any set of comma separated into a tuple if they are not otherwise specified (e.g. in a set or list).

: t = "one", "two"
: t

We can also unpack a tuple into variables in the assignment. For example, here we unpack the two variables of t into a and b.

: a, b = t
: a
: b

It's also possible to pack and define variables in the same expression. For example:

: t = a, b = "one", "two"

Note that t is only a copy of (a,b) and if we change the values of either a or b, this will not change t.

It's also possible to use the * operator to pack or unpack within method inputs or constructors. For example, the following method packs an arbitrary number of inputs into the tuple args.

: def mymethod(*args):
:     print(len(args), args)
: mymethod(0,1,2,"a","b")

Zipping

Sometimes we have multiple iterable data structures, such as lists, that we want to index through. One option is to create an index i the length of the lists and index them when iterating. However, the zip function allows us to index iterate through both at the same time:

: for i,j in zip([1,2,3],[3,2,1]):
:     print(i, j, i + j)

The method zip creates a sequence of tuples of elements at the same index, which we can unpack in the loop.

It is possible to assign zipped lists to a variable, essentially resulting in a list of tuples represneting pairs of elements of each list. We can also unzip this by expanding the list again and using zip. The outcome will be a tuple of two lists (our original input), which we unpack to variables a and b.

: el = [*zip([1,2,3],[3,2,1])]  # unpack output of zip into list 
: el
: a, b = zip(*el)  # unzip el
: a
: b

Sets

We covered the mathematical concept of sets in the tutorial and saw how we can use them in Python.

Unlike other data structures, Python sets don't allow duplicate elements. The are also unordered and we therefore cannot index them. This is because sets represent a collection, but not a sequence. We can check if an element is part of a set, and perform set operations and relation checks on them, but they are not suitable for cases where we need to index. It is possible, however, to iterate through them, but the order is not predetermined.

Operations

Review this week's tutorial for how Python sets related to their mathematical counterparts. We will recap by exploring a few operations.

For example, we can take the union of two sets using the | or or operators, which gives us a new set with all elements that are in either set.

We can also take the intersection using & or and, which gives us the set of values only in both.

: A = {'a','b','c','d'}
: B = {'c','d','e','f'}
: A | B  # union (also A or B)
: A & B  # intersection (also A and B)

We also saw in the tutorial checks such as subsets and elements. There is a lot of overlap in functionality, for example the in keyword can also be used in tuples and lists to check membership.

Dictionarys

The final common data structure in Python is the dictionary, or dict. A dictionary can be thought of as a set of "keys" that map to some value. The keys set exhibits many of the features of a set, essentially they are mutable, and contain only unique elements. Unlike a Python set, however, dictionarys do have order. However this is generally not relied upon in day-to-day use.

The values that can be contained in a dictionary may be of any time, and do not have to be unique. A dictionary is indexed using its keys. It's not possible to slice or use negative indexes (unless that negative number is a key...) to index a dictionary. Keys can be any hashable type, which includes most objects, as well as tuples, but not lists or sets.

Dictionaries are created with the {key:value} syntax, for example

: d = {'a':1, 'b':2, 'c':3}
: d
: d['a']

Dictionaries are mutable, and you can both change the elements and add new ones by indexing. If the key doesn't exist, indexing and assigning will add new key, value pair.

: d['d'] = 3

If you try to access (rather than assign to) a key that doesn't exist in the dictionary, an error will be raised.

To create an empty dictionary, you can use {}, similar to how an empty list [] and empty tuple () are created. To create an empty set, use set().

: d = {}
: d['a'] = 1
: d
Dictionary Comprehension

Like lists, tuples and sets, it's possible to create a dictionary using the comprehension mechanic.

As dictionaries require a key and value, this is often achieved using zip to iterate through two iterable data structures. For example, we create a dictionary from a set of keys and list of values.

: {k:v for k,v in zip({'a','b','c'}, [1,2,3])}
Iterating

As we don't have a numeric sequence to access, we must index the list from keys. We can access the set of keys or list of values manually, but this will decouple the two outside of the dictionary.

: set(d.keys())     # explicitly convert type
: list(d.values())  # explicitly convert type

Assessing these can be useful to see, for example, whether a key already exists, e.g. x in d.keys().

To loop through key, value pairs, we need to use d.items() which will iterate through tuples of keys and values (which we can unpack)

for k,v in d.items():
    print(k, v)

If you interate through the dictionary variable directly, you will only recieve the keys e.g.

for k in d:
    print(k, d[k])

Objected Oriented Programming

You will use the co2114_lab01.py file and the python command for this part of the worksheet. You may use IPython and import modules if you prefer.

Object oriented programming is the computer programming paradigm that organises software in a hierarchy of data and objects. Python is a language that supports the use of object object oriented programming.

Objects are conceptual data structures that have attributes, properties and methods. An instance of an object can have its own internal values, with the type of that instance being a specific "class" of object.

For a extensive tutorial on classes in Python, you can read up on it in the Python docs tutorial.

Python Classes

Python classes are the template for objects defined in Python. Classes have methods and attributes, and can be instantiated to create objects.

The core things to remember about using classes in Python are the following concepts:

  • self
  • __init__()
  • __repr__() (this is arguably less important, but we will make use of it a lot).

The self keyword is used by an object to reference itself. Because a class may have static attributes and methods, the self keyword is used to tell an object to use its own copy of attributes and methods. This allows it to take on a unique value that can be attached to a variable.

The __init__() method acts as the constructor. This is called when an object is created, and can be used to set up any specific attributes, including from arguments passed to the constructor.

The __repr__() is perhaps less common than self and __init__() but is used to define the string representation of a class. It will allow us to return a string representing how we want to view our object, specifying what information is important. This will be called when, e.g. print() is called on our object.

We define a class in Python using the class keyword, within which we can specify methods and attributes. Consider the definition of a class MyClass below:

class MyClass:
    """docstring explaining class"""
    def __init__(self, *args):
        """ constructor """
        self.args = args  # assign input args to object attribute args
    
    def __repr__(self) -> str:
        """ string representation for print() """
        return f"myclass ({self.args})"

    def myfunction(self, value) -> None:
        """ function to change object, returns nothing """
        self.value = value  # set attribute value as input value

Here we have a constructor, defined in __init__ that takes in a (arbitrary) number of input arguments, which we assign to the object's attribute args. We use tuple packing to collect the inputs.

The __repr__ method is defined to return a string representation.

We use a format string f"..." which allows us to put variables within a string using the { } characters, this will evaluate the Python expression in the braces and convert it to a string inside the outer string.

We also define the class method "myfunction" that allows us to assign a value to an abitrary attribute called value. Note that value doesn't need to be defined, or exist before we create it in this method.

In all cases, we include the keyword self, which allows us to access the current object. This should be the first argument in any object method.

Note also that I am using type identifiers here, e.g. "-> str" to indicate the output type. These are just decorators to explain to the reader what the expected output type should be. They do not enforce type, nor change the behaviour of the method.

Inheritance

One of the main features in Python is the concept of inheritance. This allows classes to be abstracted, and to create special cases of them. Consider the following set of class definitions:

class Animal:
    pass

class Bird(Animal):
    pass

class Dog(Animal):
    pass

class Duck(Bird):
    pass

Here, we have created a class Animal. We have also created a class Bird that is a subclass of Animal. This means that we can consider that any instance of a Bird is also an Animal, but not every instance of an Animal is a Bird.

We expand this further by creating a class Dog that also inherits from Animal, and a class Duck that inherits from Bird.

Using Python, we can create a Dog object and a Duck object and look at their types.

dog = Dog()
duck = Duck()

print(isinstance(dog, Dog))

print(isinstance(dog, Animal))
print(isinstance(duck, Animal))

print(isinstance(dog, Bird))
print(isinstance(duck, Bird))

The ininstance() method checks if the first argument is an object that is an instance of the second argument (which is a class type). We can see that the dog is an Animal but not a Bird, and the duck is both an Animal and Bird.

Subclasses by default inherit all methods and attributes of their superclass unless they are explitly overwritten.

Calling super()

When overwriting a method in your subclasses, by default it will not call the equivalent method in the super class. However, it is possible to call methods from the superclass using the super() method.

See below, an example where we override the constructor in our subclass. We can call the super().__init__() within the __init__() method of the subclass to execute the super constructor too.

class SuperClass:
    def __init__(self):
        print("Creating object")

class MyClass(SuperClass):
    def __init__(self, args):
        # [...]
        super().__init__()

Consider the following definitions of the data structures "queue" and "stack".

QUEUE A queue is a data structure that stores a collection of elements in order. The features of a queue are that you can either push a value to the end of the queue, or pop a value from the front. When pop is called, the value is returned and removed from the queue. This is an example of first in, first out (FIFO).

STACK A stack is a similar data structure to a queue, except that the pop method returns the value at the end of the stack. This is an example of last in, first out (LIFO).
