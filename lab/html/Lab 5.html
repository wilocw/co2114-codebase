1.2 Constraint Satisfaction Problems

A constraint satisfaction problem, or CSP, is one that is primarily focussed on assigning values to a set of variables, while satisfying some constraints.

The ConstraintSatisfactionProblem is simply the data structure that defines a CSP, defined by the variables, their domains and any constraints. In the libary provided, we have defined a class ConstraintSatisfactionProblem to encapsulate the problems we will be working with.

If you have worked through the Tutorial 5 worksheet, you will be familiar with the class.

The ConstraintSatisfactionProblem class is already implemented in constraints.csp so you don't need to implement this yourself.

class ConstraintSatisfactionProblem:
    def __init__(self, variables, constraints):
        self.variables = variables
        self.constraints = constraints
    
    @property
    def domains(self):
        return {variable: variable.domain for variable in self.variables}

    @property
    def is_consistent(self):
        return all(constraint.is_satisfied for constraint in self.constraints)
    
    @property
    def is_complete(self):
        return all(variable.is_assigned for variable in self.variables) \
                    and self.is_consistent
                    
    @property
    def arcs(self):
        return [arc for constraint in self.constraints if constraint.is_binary
                        for arc in constraint.arcs]

As you can see, we initialise a ConstraintSatisfationProblem with two things: variables and constraints, as follows:

csp = ConstraintSatisfactionProblem(variables, constraints)

The attributes we have, are

  • variables: csp.variables
  • domains: csp.domains
  • constraints: csp.domains
  • arcs: csp.arcs

We can also check the status of any assignment to see if it is consistent, with is_consistent and complete with is_complete.

1.3 Variables

One of the core components of a CSP are the set of variables, to which we must assign values.

In the library, these can be defined using the constraint.csp.Variable class:

The Variable class is already implemented in constraints.csp so you don't need to implement this yourself.

class Variable:
    def __init__(self, domain, name=None):
        self.domain = domain  # domain belongs only to one variable
        self.value = None
        self.name = name

    @property
    def is_assigned(self):
        return self.value is not None
    
    def __repr__(self):
        return str(self.value) if self.is_assigned else "?"

The Variable class is a data structure implemented to represent a variable in a constraint satisfaction problem. It has a domain that define the values that it can take on.

A Variable can be assigned any value in a domain. If a value is not None, the is_assigned property will return True. Optionally, a Variable can be given a name, which helps with printing.

1.4 Constraints and Factors

The Factor class is the data structure representing a factorised constraint.

A Factor is defined as a constraint function and the specific Variables that this constraint applies to. A summary class definition is define below:

The Factor class is already implemented in constraints.csp so you don't need to implement this yourself.

class Factor(Thing):
    def __init__(self, constraint, variables):
        self.function = constraint
        self.variables = variables

    @property
    def is_satisfied(self):
        return self.function(self.variables)
    
    @property
    def is_unary(self):
        return len(self.variables) == 1
    
    @property
    def is_binary(self):
        return len(self.variables) == 2
    
    @property
    def is_global(self):
        return len(self.variables) > 2

If it helps you, it is also possible to use the constructor Constraint() instead of Factor(). These will otherwise act exactly the same.

A Factor has variables, a collection of Variable objects, as well as a property whether it's satisfied: is_satisfied. It is also possible to check if a constraint is unary (is_unary), binary (is_binary) or global (is_global).

Replace the NotImplemented version of choice with the following code

You will revisit the choice function in Section 4

Next we will implement the main backtracking loop.

Backtracking uses recursion to try and find solutions by assigning a valid variable and continuing down the search tree until it finds that it can no longer satisfy the constraints. It will then backtrack to the previous assignment and try different values, iterating through combinations until it finds a solution that can satisfy the problem.

Replace the NotImplemented version of backtrack with the following code

def backtrack(self, csp):
    if all(variable.is_assigned for variable in csp.variables):
        return csp

    unassigned = [variable 
                    for variable in csp.variables
                        if not variable.is_assigned]

    variable = self.choice(unassigned)  # choose the next variable
    
    for value in variable.domain:  # iterate through the domain
        variable.value = value  # assign a value
        if all(constraint.is_satisfied  # check all constraints are still satisfied
                    for constraint in csp.constraints if variable in constraint): 
            print(f"{self}:\n{csp}")
            result = self.backtrack(deepcopy(csp))  # backtrack with a copy of csp
            if result: return result  # if valid, return
        variable.value = None  # else try a new value
    print(f"{self}: backtracking")
    return None  # couldn't assign a valid value

Finally we will implement the solve functionality of our agent by just calling backtrack:

Replace the NotImplemented version of solve with the following code

def solve(self, csp):
    return self.backtrack(csp)

We can add this to our environment in main() by calling:

environment.add_agent(agent)
